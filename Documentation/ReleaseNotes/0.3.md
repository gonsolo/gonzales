# Gonzales 0.3 Release Notes

## Highlights

- **Pure Swift acceleration** — CUDA/OptiX and Embree backends have been completely removed. Gonzales now relies exclusively on its native BVH with Surface Area Heuristic (SAH).
- **Modernised Swift 6.2 concurrency** — the renderer no longer depends on `@MainActor` annotations or `DispatchQueue`-based rendering. Tile rendering uses bounded structured concurrency via `TaskGroup`.
- **Cleaner public API** — `FloatX` is replaced by `Real` with self-documenting semantic aliases (`Angle`, `Distance`, `Area`, `Probability`). The central scene-loading class is renamed from `Api` to `SceneDescription`.

---

## Breaking Changes

| Before | After |
|---|---|
| `FloatX` | `Real` (+ `Angle`, `Distance`, `Area`, `Probability`) |
| `Api` / `ApiError` | `SceneDescription` / `SceneDescriptionError` |
| `Options` | `RenderConfiguration` |
| `BaseRgbSpectrum` + `RgbSpectrum` typealias | Single `RgbSpectrum` struct |
| `SurfaceInteraction.valid` flag | Return `SurfaceInteraction?` (optional) |
| `Intersectable.intersect(… interaction: inout …)` | Returns `SurfaceInteraction?` instead |
| `func spawnRay(to:)` | `func spawnRay(target:)` |
| `interaction.wo` / `interaction.uv` | `interaction.outgoing` / `interaction.uvCoordinates` |

## Removed Backends

The entire **CUDA/OptiX** bridge (`Sources/cuda`, `Sources/cudaBridge`) and the **Embree 4** module (`Sources/embree4`) have been deleted. All ray–primitive intersection is now handled by the built-in BVH accelerator.

## Concurrency Overhaul

- Removed `@MainActor` from ~40 functions and types across the codebase.
- `BoundingHierarchyBuilder.init` changed from `async` to synchronous `throws`.
- `PerspectiveCamera.init` changed from `async throws` to `throws`.
- `Boundable.worldBound` / `objectBound` changed from `async` to `throws`.
- `Camera.getSampleBounds` changed from `async` to synchronous.
- Tile rendering replaced the `DispatchQueue` + `withCheckedThrowingContinuation` pattern with a bounded `TaskGroup` that limits concurrency to `activeProcessorCount − 1`, keeping a cooperative thread free for the progress reporter.
- `@preconcurrency import Foundation` replaced with plain `import Foundation`.
- `print()` + `fflush(stdout)` replaced with `FileHandle.standardOutput.write()` for atomic output.

## Intersection Refactoring

- The dual `intersect(… tHit:)` → `Bool` / `intersect(… interaction: inout)` pattern is unified into a single method returning `SurfaceInteraction?`.
- `computeSurfaceInteraction` now takes a non-optional `TriangleIntersection` and returns an optional `SurfaceInteraction`.
- The `valid` flag on `SurfaceInteraction` is removed; optionality replaces it.

## Type System Improvements

- `FloatX` → `Real` across every source file, with semantic aliases for clarity.
- `BaseRgbSpectrum` and the `RgbSpectrum` typealias are merged into a single `RgbSpectrum` struct that directly conforms to `Spectrum`.
- `RgbSpectrum` arithmetic operators now use SIMD4 operations instead of component-wise code.
- `Spectrum` protocol gains `Sendable` conformance.
- `PiecewiseLinearSpectrum` multiplication changed from `fatalError()` to `preconditionFailure` with a message.
- `namedSpectra` changed from `@MainActor var` to `let` (immutable).

## Error Handling

- All uses of `fatalError()` and `unimplemented()` in production paths are replaced with `throw RenderError.unimplemented(…)`.
- `warning()` calls replaced with `print("Warning: …")`.
- `RenderConfiguration.makeRenderer` now accepts explicit parameters (`materials`, `acceleratorName`, `immutableState`, `renderOptions`, `meshes`) instead of reading global state.

## Variable Naming

Dozens of short or ambiguous identifiers have been renamed for readability:

| Before | After |
|---|---|
| `i`, `b`, `d`, `f`, `o` | `index`, `bucketIndex`, `diag`, `floats`, `result` |
| `a`, `b`, `c` (in `ShadingFrame`) | `valA`, `valB`, `valC` |
| `dx`, `dy` | `deltaX`, `deltaY` |
| `wo`, `wi` | `outgoing`, `incident` |
| `uv` | `uvCoordinates` |
| `mu` (Gaussian) | `mean` |
| `u` (filter sample) | `uSample` |
| `ft` (focal) | `focalT` |
| `idx` (triangle) | `triangleIndex` |
| `blaBounds` | `sahLeafBounds` |

## CI / Build

- The GitHub Actions **Build** workflow (`main.yml`) is cleaned up: comments removed, the inline `swift test` step is moved to a dedicated **Test** workflow (`test.yml`).
- `Makefile` lint target now runs SwiftLint on the entire `Sources` directory instead of only `Sources/gonzales`.
- `.swiftlint.yml` is simplified: the long single-letter identifier exclusion list is replaced with `min_length: 3` / `max_length: 40` rules; `SobolMatrices.swift` is excluded.

## Documentation

- `README.md` rewritten with feature highlights, historical Moana benchmarks (v0.0 → v0.2), Arch AUR installation instructions, and a link to the GCC 15 Swift compatibility issue.

## Removed Dead Code

- `Dummy.swift`, `FloatX.swift`, global statistics counters in `BoundingHierarchyBuilder`, `PerspectiveCamera.statistics()`, `BoundingHierarchy.statistics()`, `Camera.statistics()`, the `asyncMap` Sequence extension, and the default-material initialiser in `Options` are all deleted.
